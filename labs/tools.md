# Tools Reference

## Required Tools

- AWS account or trainer-provided sandbox account
- Web browser
- Notes application or markdown editor

## Recommended AWS Tools and Services

- IAM
- AWS CloudShell
- AWS CLI
- AWS SDKs
- AWS Lambda
- Amazon API Gateway
- Amazon DynamoDB
- Amazon S3
- Amazon SQS
- Amazon SNS
- Amazon EventBridge
- AWS Step Functions
- AWS CodeCommit or external Git provider
- AWS CodeBuild
- AWS CodeDeploy
- AWS CodePipeline
- AWS CloudFormation
- AWS SAM
- AWS Elastic Beanstalk
- Amazon ECR and Amazon ECS
- AWS Secrets Manager
- AWS Systems Manager Parameter Store
- Amazon Cognito
- AWS KMS
- Amazon CloudWatch
- AWS X-Ray
- AWS CloudTrail

## Cost Safety

- Use trainer-provided accounts where available.
- Prefer Free Tier eligible services.
- Avoid long-running compute or provisioned resources unless explicitly assigned.
- Delete Lambda functions, APIs, queues, topics, build projects, and test stacks after labs if instructed.
- Set budget alerts before using hands-on services.
- Check Billing and Cost Management after labs.

## Learner Tips

- Know which services support synchronous, asynchronous, and event-driven application patterns.
- Practice reading IAM policies and resource-based permissions.
- Keep deployment, monitoring, and rollback in mind for every application.
- Use small sample functions and test events.
- Keep a weak-topic list for exam revision.
